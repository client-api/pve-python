# Moved

Python E2E tests for `clientapi_pve` live at `e2e/` at the repo root — same
layout as the TypeScript reference (`pve-ts/e2e/`). See
[../../e2e/README.md](../../e2e/README.md) for setup and the SC-01…SC-62
scenario index.

This directory is kept only to satisfy the table in
[openapi-generator's docs](https://github.com/openapitools/openapi-generator)
that lists `tests/e2e/` as the Python convention. Add new tests under `e2e/`
at the repo root instead.
